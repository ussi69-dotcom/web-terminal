# DeckTerm Upgrade Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add gmux-inspired runtime awareness and tmux-rich workflows to DeckTerm without sacrificing its web-first strengths.

**Architecture:** Introduce a shared backend telemetry model for workspace state, render that state in browser tabs/workspaces, then layer tmux-only actions behind explicit capability gates. Extract touched session/telemetry logic into smaller modules only after the new behavior is proven.

**Tech Stack:** Bun 1.3.5+, Hono, Bun.Terminal, optional tmux backend, vanilla JS, xterm.js, Playwright, Bun test
